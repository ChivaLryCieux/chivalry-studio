# **中本聪与比特币**

## **——基于R3F的游戏化叙事三部曲**

罗锐阳 2352175 

信息可视化与数据交互叙事

## 一、网址

chivalry.studio

该网站的第九个项目为叙事入口。

## 二、概念

这个数据故事网页以“中本聪与他的比特币 18 年发展历程”为核心线索，但不是单一页面中的线性展开，而是被组织成游戏化的三个连续衔接的叙事部分。第一个部分是**叙事房间：去中心化共识18年来的野蛮生长**，以 3D 展陈空间回看中本聪、白皮书、创世区块、真实定价与价格重估；第二个部分是**徜徉于18年时间螺旋：玩家控制的3D交互叙事**，让玩家在时间螺旋中驾驶飞船穿过关键节点与数据面板；第三个部分是**棱镜之问：未来波诡云谲**，把叙事从对过去的回望推进到对未来的提问。

选择这个主题，是因为我本身正在研究这个领域。比特币并不仅仅是价格曲线或投机对象，它同时涉及密码学、分布式系统、货币形态、制度批判与技术理想主义。用数据故事和游戏化空间叙事的形式来呈现，可以把这些原本分散的线索放入同一个可视化叙事框架里，并让观者从“阅读者”转变为“穿行者”。

## 三、数据

项目中的数据主要来自两个来源：

1. 币安交易所  
   通过API获取过去 BTC 的日级价格数据，包括开盘价、最高价、最低价与收盘价，用来构建从交易所上线阶段一直延续到 2026 年 3 月底的价格曲线。

2. 比特币区块链与公开历史资料  
   用于确认创世区块、coinbase 文本、中本聪持仓估值背景以及白皮书相关的技术和历史节点。

数据收集的技术路径，是通过 Python 的 `CCXT` 库调用交易所 API 来获取历史价格数据，再整理成适合前端使用的静态数据格式。前端展示层中，价格数据、关键事件节点与 3D 场景中的叙事锚点，再分别交由 D3.js、React 和 React Three Fiber 等进行处理与可视化。

## 四、叙事与交互逻辑

这个网页不是单纯的信息堆积，而是按照“进入叙事房间 -> 穿越时间螺旋 -> 抵达未来之问”的三段式结构展开。

具体的叙事结构大致为：

1. 第一部分：叙事房间  
   这一部分以“去中心化共识18年来的野蛮生长”为主题，把中本聪、白皮书、创世区块、技术结构、真实定价与持仓象征意义组织成一个可以被环视和漫游的叙事展陈空间。它不再是传统的滚动长页，而更像一个关于匿名人物、技术发明和制度想象的数字房间。

2. 第二部分：徜徉于18年时间螺旋  
   这一部分基于 R3F 构建了 3D 交互叙事。玩家驾驶飞船在 18 年时间螺旋中穿行，沿途读取关键时间节点的数据面板，并在运动、朝向和空间距离中重新感受比特币从白皮书到共识扩张的过程。

3. 第三部分：棱镜之问  
   当玩家抵达比特币恒星之后，叙事被引向一个更抽象的问题空间：未来将走向何处？这一部分以三棱镜、光束与折射为视觉母题，把“未来波诡云谲”转化成带有象征意味的交互场景，而不是给出单一答案。

在这三部分内部，白皮书技术图、创世区块时间戳、价格重估曲线、中本聪持仓估值、时间螺旋中的关键事件数据面板，以及最终的三棱镜未来场景共同构成完整叙事闭环。

交互设计上，这个页面强调的是“叙事中的互动”，而不是工具型交互。  
交互包括：

- 叙事房间中的展板聚焦、空间浏览与进入下一段叙事的点击切换
- 时间螺旋中的飞船操控、空间位移、数据面板阅读与抵达比特币恒星后的跳转
- 三棱镜场景中的光束、折射、命中反馈与未来问题的象征性交互
- 不同叙事段落之间通过空间切换形成的节奏推进，而不只是普通滚动

## 五、技术实现

我的项目网站基于 **Next.js** 框架实现，使用了 App Router 结构来组织页面和组件。

主要技术包括：

- **Next.js**
  用于整个网页的页面结构、组件组织、路由管理和构建发布。

- **React**
  作为页面交互和组件状态管理的基础。

- **D3.js**
  早期用于 BTC 历史价格数据的可视化处理，包括时间尺度、路径生成、交互 hover 等。

- **Three.js / React Three Fiber / Drei**
  用于项目中的主要 3D 场景，包括叙事房间中的空间陈列、时间螺旋中的飞行交互叙事，以及最终三棱镜场景中的光束折射与未来象征。它们共同承担了这个项目从“网页叙事”转向“空间叙事”的关键部分。

- **pmndrs 开源生态**
  这个项目在 3D 叙事部分借助了 `pmndrs` 开源生态，包括 HDR 以及与其兼容的相关代码与工具链。它们使我能够在 React 的组件体系中组织 WebGL 场景、交互逻辑、材质效果和叙事对象，而不必完全从底层 Three.js imperative 编程开始搭建。

- **Pretext**
  早期版本使用，后来被我迭代掉了。

- **Python + CCXT**
  用于从交易所 API 中采集 BTC 历史价格数据，并进行前置清洗与整理。

- **CSS Modules**
  用于该项目页面的局部样式组织，保证叙事型页面的视觉风格统一且可控。

整体实现策略上，我将数据预处理与前端渲染分离：  
先通过 Python 获取并整理数据，再在前端通过静态数据源与 D3.js / React / R3F 进行可视化和交互呈现，从而避免页面运行时依赖外部实时接口。同时，我把“数据故事”拆分成叙事房间、时间螺旋和棱镜之问三个部分，使视觉语汇、交互模式和叙事节奏能够逐段升级。

## 六、反思

这次创作中，我最满意的部分有两个：

1. 对这个18年图景的三段式空间叙事  
   页面没有把比特币仅仅处理成价格史，而是试图通过**“叙事房间 - 时间螺旋 - 棱镜之问”**的结构，同时覆盖人物、技术、历史节点、市场重估和未来想象，让整个故事拥有更完整的纵深。

2. 对 BTC 疯狂生长过程的游戏化可视化叙事  
   价格曲线不只是数据图，而是叙事的一部分。它在时间螺旋中转化成可穿越、可接近、可读取的数据路径，使“18年的野蛮生长”不只被看到，也被体验到。

最具挑战的部分，同样也来自这两个方向：

- 一方面，要如何在宏观叙事里保持技术准确性，而不是把比特币讲成一种空泛的神话；
- 另一方面，要如何把 BTC 这条被过度熟悉的价格曲线重新做成有空间感、交互性和叙事张力的体验，而不是普通金融图表。

最终，这个项目更像一次关于“如何把技术史、金融史与未来问题写成游戏化网页叙事”的尝试。它既不是单纯的研究摘要，也不是普通的数据大屏，而是希望在视觉、交互、空间和文字之间建立一种共同推进故事的结构。
