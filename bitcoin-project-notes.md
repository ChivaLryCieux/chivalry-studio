# 神秘人中本聪与他的个人比特币项目

## 概念

这个数据故事网页以“中本聪与他的比特币 18 年发展历程”为核心线索，试图把一个通常被拆散在技术史、金融史、价格史和互联网文化中的主题，重新组织成一条连续的叙事路径。页面从中本聪与 2008 年白皮书出发，经过 2009 年创世区块、比特币价格在交易所时代的持续重估，再回到中本聪本人及其从未显著移动的大量 BTC 所构成的象征意义。

选择这个主题，是因为我本身正在研究这个领域。比特币并不仅仅是价格曲线或投机对象，它同时涉及密码学、分布式系统、货币形态、制度批判与技术理想主义。用数据故事的形式来呈现，可以把这些原本分散的线索放入同一个可视化叙事框架里。

## 数据

项目中的数据主要来自两个来源：

1. 币安交易所  
   用于获取过去 BTC 的日级价格数据，包括开盘价、最高价、最低价与收盘价，用来构建从交易所上线阶段一直延续到 2026 年 3 月底的价格曲线。

2. 比特币区块链与公开历史资料  
   用于确认创世区块、coinbase 文本铭文、中本聪持仓估值背景以及白皮书相关的技术和历史节点。

数据收集的技术路径，是通过 Python 的 `CCXT` 库调用交易所 API 来获取历史价格数据，再整理成适合前端使用的静态数据格式。前端展示层中，价格数据和关键事件节点再交由 D3.js 进行处理与可视化。

## 叙事与交互逻辑

这个网页不是单纯的信息堆积，而是按照“人物与问题提出 -> 技术基础 -> 历史启动 -> 市场重估 -> 身份与意义回望”的逻辑展开。

具体的叙事结构大致为：

1. 开场  
   以“神秘人中本聪与他的个人比特币项目”为标题，把页面定位为一个关于匿名人物、技术发明和制度想象的故事，而不是一份普通的行情展示页面。

2. 白皮书的技术基础  
   用 3D 关系图呈现白皮书中涉及的分布式网络、密码学与区块链结构，把技术逻辑可视化，而不是停留在文本说明。

3. 创世区块  
   通过关键里程碑卡片与双语“铭文”模块，强调创世区块中的那句《泰晤士报》头条既是时间戳，也是对旧金融救助机制的评论。

4. 共识扩张的时间线  
   用时间线模块串联白皮书发布、创世区块、真实定价、减半、交易所时代和机构化叙事，让读者看到比特币如何从一个极客系统逐渐扩展为全球叙事对象。

5. 价格的可视化叙事  
   用 D3.js 生成 BTC 的价格曲线，并保留 hover 交互与关键事件标注，使价格不再只是背景数据，而成为“共识扩张读数器”。

6. 中本聪持仓的意义  
   通过占比图与价值估算，把 110 万枚 BTC 的数量级和象征意义明确呈现，同时引导读者从“财富”转向“技术理性与理想主义”的理解。

7. 尾声  
   最后以“为什么中本聪不愿意公开身份？”作为总结，回到去中心化叙事与匿名性背后的思想意义。

交互设计上，这个页面强调的是“叙事中的互动”，而不是工具型交互。  
交互包括：

- 白皮书技术图中的高亮切换
- 价格曲线中的 hover 提示
- 不同模块之间通过滚动形成的叙事推进
- 视觉层次、动态漂浮与重点模块居中强化所带来的节奏感

## 技术实现

这个项目基于 **Next.js** 框架实现，使用了 App Router 结构来组织页面和组件。

主要技术包括：

- **Next.js**
  用于整个网页的页面结构、组件组织、路由管理和构建发布。

- **React**
  作为页面交互和组件状态管理的基础。

- **D3.js**
  用于 BTC 历史价格数据的可视化处理，包括时间尺度、路径生成、交互 hover 等。

- **Three.js / React Three Fiber / Drei**
  用于白皮书技术图的 3D 球体关系网络，实现空间感、轻微扰动、呼吸感和更强的视觉表现。

- **Python + CCXT**
  用于从交易所 API 中采集 BTC 历史价格数据，并进行前置清洗与整理。

- **CSS Modules**
  用于该项目页面的局部样式组织，保证叙事型页面的视觉风格统一且可控。

整体实现策略上，我将数据预处理与前端渲染分离：  
先通过 Python 获取并整理数据，再在前端通过静态数据源与 D3.js / React 进行可视化和交互呈现，从而避免页面运行时依赖外部实时接口。

## 反思

这次创作中，我最满意的部分有两个：

1. 对这个 18 年图景的宏观视野描述  
   页面没有把比特币仅仅处理成价格史，而是试图同时覆盖人物、技术、历史节点、市场重估和制度含义，让整个故事拥有更完整的纵深。

2. 对 BTC 疯狂生长过程的可视化叙事  
   价格曲线不只是数据图，而是叙事的一部分。它与创世区块、白皮书、持仓估值这些模块共同构成了一种从抽象技术到现实规模的视觉递进。

最具挑战的部分，同样也来自这两个方向：

- 一方面，要如何在宏观叙事里保持技术准确性，而不是把比特币讲成一种空泛的神话；
- 另一方面，要如何把 BTC 这条被过度熟悉的价格曲线重新做成有叙事张力的可视化，而不是普通金融图表。

最终，这个页面更像一次关于“如何把技术史与金融史写成网页叙事”的尝试。它既不是单纯的研究摘要，也不是普通的数据大屏，而是希望在视觉、交互和文字之间建立一种共同推进故事的结构。
